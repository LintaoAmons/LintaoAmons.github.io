classpath | Maven

#maven #spring

classpath 这个概念困惑了非常久，从接触到这个东西，到现在写下这篇笔记，我都是云里雾里的,现在学到Maven，基本上可以说是真正理解了。

项目结构

POM | Maven

.
├── mvnw    # --> mvn wrapper，可执行程序，保证了各个开发人员用的mvn的一致性
├── pom.xml # --> [POM | Maven](:/5b8b8eea51db43ab92ba86e743d6c321)
├── src     # --> 项目源代码的根目录
│   ├── main    # --> 生产代码(production code),最后真正上去服务器跑的代码
│   │   ├── kotlin    # --> kotlin 源代码
│   │   ├── java      # --> java 源代码
│   │   └── resources # --> 静态资源,不仅仅是 application.yml, etc.
│   └── test          # --> 测试代码的目录
│       └── kotlin    # --> kotlin 测试代码
└── target    # --> 编译打包后的产出都在这里
    ├── accountsystem-0.0.1-SNAPSHOT.jar    # --> 生成的jar包
    ├── accountsystem-0.0.1-SNAPSHOT.jar.original
    ├── classes    # --> 这里就是传说中的 classpath, 放 class 的地方, resource 包下的静态资源直接就都放这个目录下面了；这里也是代码里面，文件相对路径的起点
    │   ├── META-INF
    │   ├── application.yml    # --> resource 目录里面的 application.yml
    │   └── top    # --> 顶级 package
    ├── generated-sources
    ├── generated-test-sources
    ├── surefire-reports
    └── test-classes
        ├── META-INF
        └── top
